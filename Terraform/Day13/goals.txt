DAY 13 — PLATFORM MONITORING & OBSERVABILITY
=============================================

PRIMARY GOALS
-------------
1. Create a centralized Log Analytics Workspace via Terraform module
2. Attach diagnostic settings to Key Vault, Storage Account, VNet, and NSG
3. Route subscription-level Activity Logs into the workspace
4. Validate log ingestion using KQL queries in Azure Portal
5. Understand the difference between metrics, logs, and activity logs

LEARNING GOALS
--------------
- Understand what Log Analytics Workspace IS at the infrastructure level
- Know why diagnostic settings are separate resources in Terraform
- Understand how Azure Monitor, Diagnostics, and Log Analytics connect
- Be able to explain centralized monitoring to a hiring manager
- Know what "AuditEvent", "AllLogs", "AllMetrics" categories do

CAREER GOALS (why this matters)
---------------------------------
- EU / Finland companies are legally required to have audit logs (GDPR, NIS2)
- Most Terraform engineers never go past resource creation
- Observability separates junior from mid/senior DevOps engineers
- You will be able to answer: "How do you monitor your infrastructure?"

WHAT SUCCESS LOOKS LIKE TODAY
-------------------------------
- terraform apply runs without errors
- Log Analytics Workspace visible in Azure Portal
- Diagnostic settings attached to all 4 resources
- At least 1 KQL query returns results in Log Analytics
- You can explain every resource block you wrote — no copy/paste blindness
